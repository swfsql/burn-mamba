§Mamba-2 SSM Block — Structured State Space Duality (SSD)

This module implements the core SSD layer from the paper “Transformers are SSMs: Generalized Models and Efficient Algorithms through Structured State Space Duality” (Dao & Gu, 2024).

§The SSD Model

The SSD layer is a multi-head selective SSM. Each head processes a sequence of per_head_dim-dimensional inputs X ∈ ℝ^{sequence×per_head_dim} through the recurrence (Eq. 1–2 of the paper):

  hₜ = Āₜ hₜ₋₁ + B̄ₜ xₜ          (state update)
  yₜ = Cₜᵀ hₜ                  (output readout)

where:

  • hₜ ∈ ℝ^{state_rank×per_head_dim} is the hidden state
  • Āₜ = exp(Δₜ A) ∈ ℝ is a scalar decay (the key SSD constraint: Āₜ = αₜ · I, i.e. scalar times identity, rather than a diagonal matrix)
  • B̄ₜ = Δₜ Bₜ ∈ ℝᴺ is the (discretised) input projection
  • Cₜ ∈ ℝᴺ is the output projection
  • Δₜ > 0 is the (input-dependent) discretisation step size
  • A < 0 is a learnable scalar decay rate per head

§State Space Duality

Unrolling the recurrence yields an equivalent attention-like form (Eq. 6–7):

  M = L ∘ (C Bᵀ)      ∈ ℝ^{sequence×sequence}
  Y = M · X

where L is the 1-semiseparable mask (Eq. 4–5):

  Lᵢⱼ = āᵢ · āᵢ₋₁ · ... · āⱼ₊₁    (i ≥ j)
  Lᵢⱼ = 0                     (i < j)

This makes the layer equivalent to causal linear attention Y = (L ∘ QKᵀ) V under the renaming (C, B, X) ↦ (Q, K, V).
