Cumulative quaternion product with a custom, memory-efficient backward — the drop-in for crate::mamba3::rotation::quat_cumprod used by the Quaternion4D forward.

Routes the scan through Mamba3QuatScanBackendExt (so Autodiff gets the recompute backward), then takes final_carry = cum[:, −1] as a thin autodiff slice — its gradient folds back into cum’s gradient before the custom node runs, so the node needs only the single cum output.

Mathematically identical to quat_cumprod (asserted on values and gradients by the tests); only the backward’s memory profile differs.

§Shapes

  • q_bshj4 : [batch, sequence, nheads, blocks, 4] — per-step unit quaternions (the recompute backward’s gradient identities assume unit norm, which holds for quat_from_scaled_axis outputs and products thereof).
  • init : optional carry [batch, nheads, blocks, 4] (identity if None).
  • returns (cum, final_carry) — [batch, sequence, nheads, blocks, 4] and [batch, nheads, blocks, 4].
