1//! # Quaternion (k=4) rotational state — the non-abelian generalisation of RoPE
2//!
3//! Mamba-3's data-dependent RoPE realises a **complex-valued** SSM: the state
4//! transition factors as a per-head scalar decay times a block-diagonal of
5//! `2×2` rotations (paper Prop. *Complex-to-Real SSM Equivalence*), and because
6//! `SO(2) ≅ U(1)` is **abelian** the cumulative rotation collapses to a
7//! `cumsum` of angles and is absorbed into `B`/`C` (the "RoPE trick", Prop.
8//! *Complex SSM, Data-Dependent RoPE Equivalence*).  See
9//! [`crate::mamba3::double_ssd::double_ssd::apply_rope`].
10//!
11//! This module implements the next rung of the ladder: a **quaternion**
12//! (`k = 4`) rotational state, i.e. the transition's rotation lives in the
13//! left-isoclinic subgroup `SU(2) ⊂ SO(4)` instead of `SO(2)`.  Unit
14//! quaternions under multiplication are `SU(2)`, which is **non-abelian** and
15//! contains non-solvable finite subgroups (the binary icosahedral group
16//! `2I = SL(2,5)`, a double cover of `A₅`).  By Barrington's theorem this lifts
17//! the layer's reachable state-tracking from the solvable/`TC⁰` regime (parity,
18//! mod-k) toward `NC¹`, which abelian rotations provably cannot reach.
19//!
20//! ## What survives, what changes
21//!
22//! The key fact (derivable purely from telescoping + orthogonality, **without**
23//! commutativity — see the crate discussion) is that the RoPE *factoring*
24//! survives intact: with the **ordered** cumulative rotation
25//! `Pₜ = Rₜ Rₜ₋₁ ⋯ R₁`,
26//!
27//! ```text
28//!   Cₜᵀ (Rₜ⋯Rᵢ₊₁) Bᵢ  =  (Pₜᵀ Cₜ)ᵀ (Pᵢᵀ Bᵢ)  =  C̄ₜᵀ B̄ᵢ ,
29//! ```
30//!
31//! so the scalar-decay SSD core (`L ⊙ C̄B̄ᵀ`) is **unchanged** — only the
32//! projections `B̄ᵢ = Pᵢᵀ Bᵢ`, `C̄ₜ = Pₜᵀ Cₜ` are rotated.  What is lost is the
33//! closed-form `cumsum`: the cumulative rotation must be built by an
34//! **associative scan over the per-step quaternions** ([`quat_cumprod`]) rather
35//! than a sum of angles.  Because a product of unit quaternions is again a unit
36//! quaternion, the scan stays exactly orthogonal (no drift, no `wrap_angle`
37//! needed), and the cross-chunk carry is a single quaternion per block/head —
38//! the exact analogue of `cum_angle` in the existing caches.
39//!
40//! `SO(2)` (today's `apply_rope`) is the abelian collapse: restricting each
41//! quaternion to a single fixed axis makes them commute and reduces
42//! [`quat_cumprod`] to a `cumsum` of half-angles (asserted in the tests).
43//!
44//! ## Pipeline (the `k = 4` instantiation of the rotation block)
45//!
46//! ```text
47//!   per-step unit quaternion qₜ      (materialise from the in-projection; caller)
48//!        │  quat_cumprod (assoc. scan, + cross-chunk carry)
49//!        ▼
50//!   cumulative rotation Qₜ
51//!        │  rotate_state_rank_blocks(B, conj(Qₜ)) , rotate_state_rank_blocks(C, conj(Qₜ))
52//!        ▼
53//!   B̄, C̄  ──►  standard scalar-decay SSD  (unchanged)
54//! ```
55//!
56//! Quaternion layout: the last axis has size 4 and holds `(w, x, y, z)` with
57//! `w` the real part.  A `state_rank` of `r = 4·J` is treated as `J` independent
58//! quaternion blocks; the rotation acts within each block, exactly as RoPE acts
59//! within each `2`-pair.  This module is a self-contained, tested reference for
60//! the math; wiring it into the [`Mamba3`](crate::mamba3::mamba3::Mamba3) block
61//! is a separate, larger change (the SSD kernels themselves need no edits).

71/// Which rotational-state algebra the block uses for the data-dependent
72/// positional rotation of `B`/`C`.
73///
74/// - [`Complex2D`](RotationKind::Complex2D) — the abelian `SO(2)`/complex RoPE
75///   that Mamba-3 ships: cumulative *angles* via `cumsum`, applied by
76///   [`apply_rope`]. The default; behaviourally unchanged.
77/// - [`Quaternion4D`](RotationKind::Quaternion4D) — the non-abelian
78///   `SU(2) ⊂ SO(4)` quaternion rotation of this module: cumulative *product*
79///   via [`quat_cumprod`], applied by [`rotate_state_rank_blocks`]. Richer
80///   state-tracking; selects the [`RotationState::Quaternion`] cache accumulator.
81#[derive(Debug, Clone, Copy, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
82pub enum RotationKind {
83    /// Abelian complex (`SO(2)`) RoPE — the current default behaviour.
84    Complex2D,
85    /// Non-abelian quaternion (`SU(2)`) rotation.
86    Quaternion4D,
87}
88
89impl Default for RotationKind {
90    fn default() -> Self {
91        RotationKind::Complex2D
92    }
93}

95/// The cumulative-rotation accumulator carried between calls in a Mamba-3 cache
96/// — the variant matching the block's [`RotationKind`].
97///
98/// - [`Angle`](RotationState::Angle) — abelian per-pair cumulative RoPE angle,
99///   shape `[batch, nheads, num_rope_angles]` (today's `cum_angle`).
100/// - [`Quaternion`](RotationState::Quaternion) — per-block cumulative unit
101///   quaternion, shape `[batch, nheads, blocks, 4]`, produced by
102///   [`quat_cumprod`].
103///
104/// This is the cache-level counterpart of [`RotationKind`]. It is defined here
105/// (the rotation module owns the accumulator type); substituting it for the
106/// pathway caches' `cum_angle_bha` field happens together with the forward/step
107/// wiring that consumes it.
108#[derive(Module, Debug)]
109pub enum RotationState {
110    /// Abelian RoPE cumulative angle, shape `[batch, nheads, num_rope_angles]`.
111    Angle(Tensor<3>),
112    /// Quaternion cumulative rotation, shape `[batch, nheads, blocks, 4]`.
113    Quaternion(Tensor<4>),
114}
115
116impl RotationState {
117    /// Zero-initialised abelian angle accumulator `[batch, nheads, num_rope_angles]`.
118    pub fn zeros_angle(
119        batch: usize,
120        nheads: usize,
121        num_rope_angles: usize,
122        device: &Device,
123    ) -> Self {
124        RotationState::Angle(Tensor::zeros([batch, nheads, num_rope_angles], device))
125    }
126
127    /// Identity-initialised quaternion accumulator `[batch, nheads, blocks, 4]`
128    /// (every block is the identity quaternion `(1, 0, 0, 0)`).
129    pub fn identity_quaternion(
130        batch: usize,
131        nheads: usize,
132        blocks: usize,
133        device: &Device,
134    ) -> Self {
135        let w = Tensor::ones([batch, nheads, blocks, 1], device);
136        let xyz = Tensor::zeros([batch, nheads, blocks, 3], device);
137        RotationState::Quaternion(Tensor::cat(vec![w, xyz], 3))
138    }
139
140    /// Unwrap the abelian angle accumulator; panics if this is a quaternion.
141    pub fn angle(self) -> Tensor<3> {
142        match self {
143            RotationState::Angle(a) => a,
144            RotationState::Quaternion(_) => {
145                panic!("RotationState is Quaternion, expected Angle")
146            }
147        }
148    }
149
150    /// Unwrap the quaternion accumulator; panics if this is an angle.
151    pub fn quaternion(self) -> Tensor<4> {
152        match self {
153            RotationState::Quaternion(q) => q,
154            RotationState::Angle(_) => panic!("RotationState is Angle, expected Quaternion"),
155        }
156    }
157
158    /// Run the [`NaN`/`Inf` guards](crate::utils::sanity) on the held tensor.
159    pub fn sanity(&self) {
160        match self {
161            RotationState::Angle(a) => crate::utils::sanity::sanity(a),
162            RotationState::Quaternion(q) => crate::utils::sanity::sanity(q),
163        }
164    }
165}

167// ---------------------------------------------------------------------------
168// Quaternion algebra on the trailing `(w, x, y, z)` axis
169// ---------------------------------------------------------------------------
170
171/// Hamilton product `a ⊗ b` of two quaternion tensors.
172///
173/// Both inputs have shape `[..., 4]` with the last axis ordered `(w, x, y, z)`;
174/// the product is computed component-wise and broadcasts over the leading dims.
175/// Quaternion multiplication is **non-commutative** (`a ⊗ b ≠ b ⊗ a` in
176/// general) but associative.
177///
178/// Identifying `ℝ⁴` with the quaternions, left-multiplication `v ↦ a ⊗ v` is
179/// exactly the action of the `4×4` rotation matrix [`quat_to_rot4`]`(a)`, so
180/// this is also how a rotation is *applied* to a state/`B`/`C` block (see
181/// [`rotate_state_rank_blocks`]).
182pub fn quat_mul<const D: usize>(a: Tensor<D>, b: Tensor<D>) -> Tensor<D> {
183    let n = D - 1;
184    let aw = a.clone().narrow(n, 0, 1);
185    let ax = a.clone().narrow(n, 1, 1);
186    let ay = a.clone().narrow(n, 2, 1);
187    let az = a.narrow(n, 3, 1);
188    let bw = b.clone().narrow(n, 0, 1);
189    let bx = b.clone().narrow(n, 1, 1);
190    let by = b.clone().narrow(n, 2, 1);
191    let bz = b.narrow(n, 3, 1);
192
193    // Hamilton product (each term is shape [..., 1]).
194    let w = aw.clone() * bw.clone() - ax.clone() * bx.clone() - ay.clone() * by.clone()
195        - az.clone() * bz.clone();
196    let x = aw.clone() * bx.clone() + ax.clone() * bw.clone() + ay.clone() * bz.clone()
197        - az.clone() * by.clone();
198    let y = aw.clone() * by.clone() - ax.clone() * bz.clone() + ay.clone() * bw.clone()
199        + az.clone() * bx.clone();
200    let z = aw * bz + ax * by - ay * bx + az * bw;
201
202    Tensor::cat(vec![w, x, y, z], n)
203}

205/// Quaternion conjugate `q* = (w, −x, −y, −z)` (shape `[..., 4]`).
206///
207/// For a **unit** quaternion `q* = q⁻¹`, and the corresponding rotation matrix
208/// satisfies `Lₚ⋆ = Lₚᵀ = Lₚ⁻¹`.  Hence rotating by the *inverse* cumulative
209/// rotation (`B̄ = Pᵀ B`) is `rotate_state_rank_blocks(B, conj(Q))`.
210pub fn quat_conj<const D: usize>(q: Tensor<D>) -> Tensor<D> {
211    let n = D - 1;
212    let w = q.clone().narrow(n, 0, 1);
213    let xyz = q.narrow(n, 1, 3);
214    Tensor::cat(vec![w, -xyz], n)
215}

217/// Normalise quaternions to unit norm along the last axis (shape `[..., 4]`).
218///
219/// The per-step rotation is materialised from a raw, unconstrained projection
220/// and normalised here so it is a genuine unit quaternion (an element of
221/// `SU(2)`), the analogue of `tanh(θ)·π` bounding the RoPE angle.  A tiny floor
222/// guards the zero-quaternion.
223pub fn quat_normalize<const D: usize>(q: Tensor<D>) -> Tensor<D> {
224    let n = D - 1;
225    let norm = (q.clone() * q.clone()).sum_dim(n).sqrt().clamp_min(1e-12);
226    q / norm
227}
228
229/// Materialise a unit quaternion from a **scaled rotation vector** `g ∈ ℝ³`
230/// (axis · angle) via the exponential map — the data-dependent "materialise
231/// `Rₜ`" step, analogous to RoPE's `Δₜ · π · tanh(θₜ)` angle.
232///
233/// With `‖g‖ = angle` and `ĝ = g / angle` the axis, returns the unit quaternion
234/// `q = (cos(angle/2), sin(angle/2)·ĝ)`.  A vanishing `g` maps to the identity
235/// `(1, 0, 0, 0)`, so scaling `g` by a small `Δₜ` (the discretisation step)
236/// yields a near-identity rotation — exactly the regime where a small step
237/// barely rotates the state.  The `sin(angle/2)/angle` factor is the numerically
238/// stable form of the (otherwise `0/0`) per-component scale near `g = 0`.
239///
240/// # Shapes
241/// - `g` : `[..., 3]`
242/// - out : `[..., 4]` (ordered `(w, x, y, z)`), unit norm.
243pub fn quat_from_scaled_axis<const D: usize>(g: Tensor<D>) -> Tensor<D> {
244    let n = D - 1;
245    let angle = (g.clone() * g.clone()).sum_dim(n).sqrt(); // [..., 1]
246    let half = angle.clone() * 0.5;
247    let w = half.clone().cos(); // [..., 1]
248    // sin(angle/2) / angle  → 1/2 as angle → 0 (no rotation), guarded against 0/0.
249    let scale = half.sin() / angle.clamp_min(1e-12); // [..., 1]
250    let v = g * scale; // [..., 3]
251    quat_normalize(Tensor::cat(vec![w, v], n))
252}

254/// Materialise the `4×4` orthogonal matrix of left-multiplication by `q`.
255///
256/// Maps `q` of shape `[..., 4]` to `[..., 4, 4]` such that, for `v` of shape
257/// `[..., 4]`, `Lq · v == quat_mul(q, v)`.  Concretely (rows = output coords,
258/// cols = input coords, all in `(w, x, y, z)` order):
259///
260/// ```text
261///   ⎡ w  -x  -y  -z ⎤
262///   ⎢ x   w  -z   y ⎥
263///   ⎢ y   z   w  -x ⎥
264///   ⎣ z  -y   x   w ⎦
265/// ```
266///
267/// For a unit `q` this is orthogonal with `det = 1` (a left-isoclinic rotation).
268/// Provided mainly for the generic / verification path; the cheap way to apply a
269/// rotation is [`rotate_state_rank_blocks`] (a quaternion product, no `4×4`
270/// materialisation).  `DR` must equal `D + 1`.
271pub fn quat_to_rot4<const D: usize, const DR: usize>(q: Tensor<D>) -> Tensor<DR> {
272    assert_eq!(D + 1, DR, "quat_to_rot4 maps rank D to rank D+1");
273    let n = D - 1;
274    let w = q.clone().narrow(n, 0, 1);
275    let x = q.clone().narrow(n, 1, 1);
276    let y = q.clone().narrow(n, 2, 1);
277    let z = q.narrow(n, 3, 1);
278
279    // Each row is a [..., 4] tensor (the four column entries).
280    let row0 = Tensor::cat(vec![w.clone(), -x.clone(), -y.clone(), -z.clone()], n);
281    let row1 = Tensor::cat(vec![x.clone(), w.clone(), -z.clone(), y.clone()], n);
282    let row2 = Tensor::cat(vec![y.clone(), z.clone(), w.clone(), -x.clone()], n);
283    let row3 = Tensor::cat(vec![z, -y, x, w], n);
284
285    // Stack the rows along a freshly inserted row axis → [..., 4, 4].
286    Tensor::cat(
287        vec![
288            row0.unsqueeze_dim::<DR>(n),
289            row1.unsqueeze_dim::<DR>(n),
290            row2.unsqueeze_dim::<DR>(n),
291            row3.unsqueeze_dim::<DR>(n),
292        ],
293        n,
294    )
295}

297// ---------------------------------------------------------------------------
298// Rotation application on the state_rank axis
299// ---------------------------------------------------------------------------
300
301/// Apply a per-block quaternion rotation to the `state_rank` axis of `v`.
302///
303/// `v` has shape `[..., state_rank]` with `state_rank = 4·J`, viewed as `J`
304/// independent quaternion blocks; `q` has shape `[..., J, 4]` (one unit
305/// quaternion per block, same leading dims as `v`).  Returns `q ⊗ v` per block,
306/// i.e. the rotation `L_q` applied within each `4`-block, reshaped back to
307/// `[..., state_rank]`.
308///
309/// This is the generalisation of RoPE's per-pair `2×2` rotation to per-block
310/// `4×4`.  To rotate by the *inverse* cumulative rotation when absorbing into
311/// `B`/`C` (`B̄ = Pᵀ B`), pass `q = conj(Qcum)`:
312/// `rotate_state_rank_blocks(b, conj(qcum))`.
313///
314/// `DB` must equal `D + 1` (the block-split inserts the `J` axis).
315pub fn rotate_state_rank_blocks<const D: usize, const DB: usize>(
316    v: Tensor<D>,
317    q: Tensor<DB>,
318) -> Tensor<D> {
319    assert_eq!(D + 1, DB, "rotate_state_rank_blocks splits one axis into (J, 4)");
320    let dims = v.dims();
321    let state_rank = dims[D - 1];
322    assert_eq!(state_rank % 4, 0, "state_rank must be a multiple of 4 (quaternion blocks)");
323    let blocks = state_rank / 4;
324
325    // Build the block-split shape [..., J, 4] (rank DB) and the flat shape
326    // [..., state_rank] (rank D) for the round trip.
327    let mut split_shape = [0usize; DB];
328    split_shape[..D - 1].copy_from_slice(&dims[..D - 1]);
329    split_shape[DB - 2] = blocks;
330    split_shape[DB - 1] = 4;
331
332    let v_blocks = v.reshape(split_shape); // [..., J, 4]
333    let rotated = quat_mul(q, v_blocks); // L_q applied per block
334    rotated.reshape(dims) // [..., state_rank]
335}

337// ---------------------------------------------------------------------------
338// Cumulative rotation scan (the associative, non-abelian replacement for cumsum)
339// ---------------------------------------------------------------------------
340
341/// Cumulative (ordered, left-accumulating) quaternion product along the
342/// sequence axis, with a cross-chunk carry.
343///
344/// This is the non-abelian analogue of the cumulative *sum of angles* used by
345/// RoPE: where complex rotations compose by adding angles (a `cumsum`),
346/// quaternions compose by multiplication, which is order-dependent, so a real
347/// scan is required.
348///
349/// # Shapes
350/// - `q_bshj4` : `[batch, sequence, nheads, J, 4]` per-step **unit** quaternions
351///   (block count `J = state_rank / 4`).
352/// - `init`    : optional carry `[batch, nheads, J, 4]` — the cumulative
353///   rotation at the end of the previous chunk (identity `(1,0,0,0)` for a fresh
354///   start).
355/// - returns `(cum, final_carry)` where `cum` is `[batch, sequence, nheads, J, 4]`
356///   with `cum[:, t] = qₜ ⊗ qₜ₋₁ ⊗ ⋯ ⊗ q₀ ⊗ init` (newest on the left, matching
357///   `Pₜ = Rₜ ⋯ R₁`), and `final_carry` `[batch, nheads, J, 4]` is `cum[:, −1]`
358///   to thread into the next chunk.
359///
360/// Running this over a split sequence while threading `final_carry` is exactly
361/// equal to running it over the whole sequence (asserted in the tests) — the
362/// chunked-prefill / streaming guarantee, here for the rotation accumulator.
363///
364/// Implemented as a **Hillis–Steele** inclusive associative scan: the quaternion
365/// product is associative (just not commutative), so a log-depth scan applies as
366/// long as operand order is preserved (newest-on-left). Each doubling step is a
367/// single full-tensor [`quat_mul`] plus a sequence shift, so the *sequential
368/// dependency depth* is `O(log sequence)` rather than the `O(sequence)` of a
369/// token-by-token loop — the same values, but a handful of large batched kernels
370/// instead of thousands of serialized tiny ones (and a correspondingly shallow
371/// autodiff graph). The sequential reference it replaces is kept as a test oracle
372/// (`quat_cumprod_sequential` in the tests module) and asserted equal on values
373/// **and** gradients.
374pub fn quat_cumprod(q_bshj4: Tensor<5>, init: Option<Tensor<4>>) -> (Tensor<5>, Tensor<4>) {
375    let [batch, sequence, nheads, blocks, _four] = q_bshj4.dims();
376    let device = q_bshj4.device();
377
378    // Pure prefix product Pₜ = qₜ ⊗ qₜ₋₁ ⊗ ⋯ ⊗ q₀ by Hillis–Steele doubling.
379    // Invariant after each step with offset `d`: a[t] holds the product of the
380    // window [t .. max(t-2d+1, 0)] (newest on the left). After ⌈log₂ sequence⌉
381    // doublings the window covers [t .. 0], i.e. a[t] = Pₜ.
382    let mut a = q_bshj4;
383    let mut offset = 1usize;
384    while offset < sequence {
385        // shifted[t] = a[t-offset] for t ≥ offset, else the identity quaternion
386        // (1,0,0,0) — so the first `offset` prefixes pass through unchanged
387        // (a ⊗ identity = a).
388        let ident = {
389            let w = Tensor::ones([batch, offset, nheads, blocks, 1], &device);
390            let xyz = Tensor::zeros([batch, offset, nheads, blocks, 3], &device);
391            Tensor::cat(vec![w, xyz], 4)
392        };
393        let shifted = Tensor::cat(vec![ident, a.clone()], 1).narrow(1, 0, sequence);
394        // Recent block (a) on the left, older block (shifted) on the right.
395        a = quat_mul(a, shifted);
396        offset *= 2;
397    }
398
399    // Fold the cross-chunk carry once: cumₜ = Pₜ ⊗ init. `init` (the previous
400    // chunk's final cumulative rotation) is the oldest factor, hence on the
401    // right; a missing carry is the identity and needs no multiply.
402    let cum = match init {
403        Some(init_bhj4) => {
404            assert_eq!([batch, nheads, blocks, 4], init_bhj4.dims());
405            quat_mul(a, init_bhj4.unsqueeze_dim::<5>(1)) // [batch, 1, nheads, J, 4] broadcasts over seq
406        }
407        None => a,
408    };
409
410    let final_carry = cum.clone().narrow(1, sequence - 1, 1).squeeze_dim::<4>(1); // [batch, nheads, J, 4]
411    (cum, final_carry)
412}

414// ---------------------------------------------------------------------------
415// Partial block rotation (rope_fraction support)
416// ---------------------------------------------------------------------------
417
418/// Apply a per-block quaternion rotation to the first `rope_width` entries of
419/// the `state_rank` axis (a multiple of 4); the remainder passes through. The
420/// quaternion analogue of [`apply_rope_partial`].
421///
422/// `q` has one quaternion per rotated block (`rope_width / 4` of them). `DB`
423/// must equal `D + 1`.
424pub fn rotate_blocks_partial<const D: usize, const DB: usize>(
425    v: Tensor<D>,
426    q: Tensor<DB>,
427    rope_width: usize,
428) -> Tensor<D> {
429    let r = v.dims()[D - 1];
430    if rope_width == r {
431        rotate_state_rank_blocks::<D, DB>(v, q)
432    } else {
433        let head = v.clone().narrow(D - 1, 0, rope_width);
434        let tail = v.narrow(D - 1, rope_width, r - rope_width);
435        let head_rot = rotate_state_rank_blocks::<D, DB>(head, q);
436        Tensor::cat(vec![head_rot, tail], D - 1)
437    }
438}

441// Forward / step rotation of B and C (shared by both SSD pathways)
442// ---------------------------------------------------------------------------
443
444/// Rotate `B`/`C` for a **full sequence** by the data-dependent positional
445/// rotation, returning the rotated projections and the new cumulative
446/// [`RotationState`] to store in the cache.
447///
448/// Branches on [`RotationKind`]:
449/// - [`Complex2D`](RotationKind::Complex2D): the abelian RoPE — cumulative
450///   angle `cumsum` continued from `prev`, then [`apply_rope_partial`]. Exactly
451///   the original Mamba-3 behaviour.
452/// - [`Quaternion4D`](RotationKind::Quaternion4D): per-step unit quaternion
453///   [`quat_from_scaled_axis`] (the in-projection generators scaled per-head by
454///   `Δ`), composed by [`quat_cumprod`] continuing the cached quaternion, then
455///   applied to `B`/`C` as `rotate(·, conj(Qₜ))` over the first `4·blocks`
456///   state-rank entries.
457///
458/// # Shapes
459/// - `rot_bsa` : `[batch, sequence, num_rotation_channels]` — the in-projection
460///   rotation channels (angles for Complex2D, `3·blocks` quaternion generators
461///   for Quaternion4D).
462/// - `dt_bsh`  : `[batch, sequence, nheads]` (`Δ`).
463/// - `b_bsmhr` / `c_bsmhr` : `[batch, sequence, mimo_rank, nheads, state_rank]`.
464pub fn rotate_bc_forward(
465    rot_bsa: Tensor<3>,
466    dt_bsh: Tensor<3>,
467    prev: RotationState,
468    b_bsmhr: Tensor<5>,
469    c_bsmhr: Tensor<5>,
470    kind: RotationKind,
471    rope_dim: usize,
472) -> (Tensor<5>, Tensor<5>, RotationState) {
473    let [batch, sequence, mimo_rank, nheads, _state_rank] = b_bsmhr.dims();
474    match kind {
475        RotationKind::Complex2D => {
476            let prev_angle_bha = prev.angle();
477            let num_rope_angles = prev_angle_bha.dims()[2];
478            let theta_scaled_bsa = rot_bsa.tanh() * std::f32::consts::PI;
479            let raw_angles_bsha =
480                dt_bsh.unsqueeze_dim::<4>(3) * theta_scaled_bsa.unsqueeze_dim::<4>(2);
481            let cum_angles_bsha = prev_angle_bha.unsqueeze_dim::<4>(1) + raw_angles_bsha.cumsum(1);
482            let cum_angles_bsmha = cum_angles_bsha
483                .clone()
484                .unsqueeze_dim::<5>(2)
485                .expand([batch, sequence, mimo_rank, nheads, num_rope_angles]);
486            let rotate_pairwise = mimo_rank == 1;
487            let b = apply_rope_partial::<5>(b_bsmhr, cum_angles_bsmha.clone(), rope_dim, rotate_pairwise);
488            let c = apply_rope_partial::<5>(c_bsmhr, cum_angles_bsmha, rope_dim, rotate_pairwise);
489            let last = wrap_angle(cum_angles_bsha.narrow(1, sequence - 1, 1).squeeze_dim::<3>(1));
490            (b, c, RotationState::Angle(last))
491        }
492        RotationKind::Quaternion4D => {
493            let prev_q_bhj4 = prev.quaternion();
494            let blocks = prev_q_bhj4.dims()[2];
495            let rope_width = blocks * 4;
496            // Generators [b,s,blocks,3] (shared across heads), scaled per-head by Δ.
497            let g_bshj3 = rot_bsa.reshape([batch, sequence, blocks, 3]).unsqueeze_dim::<5>(2)
498                * dt_bsh.unsqueeze_dim::<4>(3).unsqueeze_dim::<5>(4);
499            let q_step_bshj4 = quat_from_scaled_axis::<5>(g_bshj3);
500            // Memory-efficient scan: a custom recompute backward (saves only the
501            // leaf inputs) instead of retaining the scan's intermediates. Equal
502            // to [`quat_cumprod`] on values and gradients (asserted in tests).
503            let (cum_bshj4, final_bhj4) =
504                crate::mamba3::quat_scan::quat_cumprod_recalculated(q_step_bshj4, Some(prev_q_bhj4));
505            // B̄ = rotate by the inverse cumulative rotation (conjugate), per block,
506            // broadcast over the mimo_rank axis.
507            let conj_bsmhj4 = quat_conj(cum_bshj4)
508                .unsqueeze_dim::<6>(2)
509                .expand([batch, sequence, mimo_rank, nheads, blocks, 4]);
510            let b = rotate_blocks_partial::<5, 6>(b_bsmhr, conj_bsmhj4.clone(), rope_width);
511            let c = rotate_blocks_partial::<5, 6>(c_bsmhr, conj_bsmhj4, rope_width);
512            (b, c, RotationState::Quaternion(quat_normalize(final_bhj4)))
513        }
514    }
515}

517/// Single-token counterpart of [`rotate_bc_forward`] for the recurrent `step`.
518///
519/// # Shapes
520/// - `rot_ba`  : `[batch, num_rotation_channels]`.
521/// - `dt_bh`   : `[batch, nheads]`.
522/// - `b_bmhr` / `c_bmhr` : `[batch, mimo_rank, nheads, state_rank]`.
523pub fn rotate_bc_step(
524    rot_ba: Tensor<2>,
525    dt_bh: Tensor<2>,
526    prev: RotationState,
527    b_bmhr: Tensor<4>,
528    c_bmhr: Tensor<4>,
529    kind: RotationKind,
530    rope_dim: usize,
531) -> (Tensor<4>, Tensor<4>, RotationState) {
532    let [batch, mimo_rank, nheads, _state_rank] = b_bmhr.dims();
533    match kind {
534        RotationKind::Complex2D => {
535            let prev_angle_bha = prev.angle();
536            let num_rope_angles = prev_angle_bha.dims()[2];
537            let theta_scaled_ba = rot_ba.tanh() * std::f32::consts::PI;
538            let raw_angle_bha =
539                dt_bh.unsqueeze_dim::<3>(2) * theta_scaled_ba.unsqueeze_dim::<3>(1);
540            let new_cum_angle_bha = wrap_angle(prev_angle_bha + raw_angle_bha);
541            let new_cum_angle_bmha = new_cum_angle_bha
542                .clone()
543                .unsqueeze_dim::<4>(1)
544                .expand([batch, mimo_rank, nheads, num_rope_angles]);
545            let rotate_pairwise = mimo_rank == 1;
546            let b = apply_rope_partial::<4>(b_bmhr, new_cum_angle_bmha.clone(), rope_dim, rotate_pairwise);
547            let c = apply_rope_partial::<4>(c_bmhr, new_cum_angle_bmha, rope_dim, rotate_pairwise);
548            (b, c, RotationState::Angle(new_cum_angle_bha))
549        }
550        RotationKind::Quaternion4D => {
551            let prev_q_bhj4 = prev.quaternion();
552            let blocks = prev_q_bhj4.dims()[2];
553            let rope_width = blocks * 4;
554            let g_bhj3 = rot_ba.reshape([batch, blocks, 3]).unsqueeze_dim::<4>(1)
555                * dt_bh.unsqueeze_dim::<3>(2).unsqueeze_dim::<4>(3);
556            let q_step_bhj4 = quat_from_scaled_axis::<4>(g_bhj3);
557            // Single step: Qₜ = qₜ ⊗ Qₜ₋₁.
558            let new_q_bhj4 = quat_normalize(quat_mul(q_step_bhj4, prev_q_bhj4));
559            let conj_bmhj4 = quat_conj(new_q_bhj4.clone())
560                .unsqueeze_dim::<5>(1)
561                .expand([batch, mimo_rank, nheads, blocks, 4]);
562            let b = rotate_blocks_partial::<4, 5>(b_bmhr, conj_bmhj4.clone(), rope_width);
563            let c = rotate_blocks_partial::<4, 5>(c_bmhr, conj_bmhj4, rope_width);
564            (b, c, RotationState::Quaternion(new_q_bhj4))
565        }
566    }
567}
